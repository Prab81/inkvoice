## [2026-07-09] (33)
- Fixed dictation hotkey leaking into Notepad (confirmed via screenshot: Notepad's Alt-key KeyTip badges appeared, proving a raw Alt reached it even though our hotkey is Ctrl+Shift-only). Root cause: two keyboard layouts installed (en-AU, en-GB) with Windows' per-app-window input method switching — RegisterHotKey's OEM-punctuation matching (backtick) is resolved against the foreground thread's active layout at keypress time, not registration time, so the grave/backtick key stopped matching our registration and the raw keystroke fell straight through to Notepad, immediately, every time. Verified live: shell now registers Ctrl+Shift+F9 with no fallback.
